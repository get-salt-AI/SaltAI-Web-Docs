# Table Info

<div style="display: flex; gap: 20px; align-items: flex-start; margin-bottom: 20px;">
<div style="flex: 1; min-width: 0;">

Retrieves structural information for a specified MySQL table. It returns both a human-readable summary and a JSON payload describing columns, data types, nullability, and related metadata, using provided credentials and a backend service call.

</div>

## Usage

Use this node when you need to inspect or validate the schema of a MySQL table before writing queries, designing transformations, or building data pipelines. Place it early in a workflow to confirm column names, data types, and keys so downstream nodes can safely rely on the table structure.

## Important Notes
- **Credentials required**: The credentials_path input must point to valid MySQL credentials that the service can load and use for authentication.
- **Database and table must exist**: The specified database and table_name must be present and accessible to the configured database user.
- **Timeout handling**: Operations exceeding the timeout value will fail; increase the timeout for large schemas or slower database connections.
- **Primary outputs**: Downstream workflows should primarily consume the text and json outputs, as html, xlsx, and pdf are typically empty for this operation.
- **Network dependency**: This node relies on a backend service to fetch metadata, so stable network connectivity to that service is required.

## Troubleshooting
- **Authentication error**: Check that credentials_path is correct, readable, and points to a configuration with valid MySQL credentials.
- **Table not found**: Verify that table_name and database are correctly spelled, exist on the server, and match any case sensitivity rules.
- **Insufficient privileges**: Ensure the database user has permission to read metadata for the specified database and table.
- **Timeout or connectivity issues**: Increase the timeout value and verify network connectivity, firewall rules, and service availability.
- **Downstream JSON parsing failures**: If other nodes cannot parse the json output, inspect the raw json string for embedded error messages or unexpected structure from the backend.
