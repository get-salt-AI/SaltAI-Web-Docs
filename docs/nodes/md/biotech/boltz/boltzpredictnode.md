# Boltz Predict

<div style="display: flex; gap: 20px; align-items: flex-start; margin-bottom: 20px;">
<div style="flex: 1; min-width: 0;">

Runs Boltz structure prediction from a prepared Boltz YAML configuration and its auxiliary files. Produces ranked 3D structures along with confidence and scoring data. When affinity prediction is requested and valid ligand inputs are present, it also returns affinity estimates and associated confidence matrices (PAE, PDE, pLDDT) as JSON-decoded arrays.

## Usage

Use this node as the final prediction step after you have built all Boltz components (sequences, templates, constraints, properties) and combined them into a single configuration. First, assemble components with the Boltz builder nodes, merge them with Boltz List Combiner and Boltz YAML Combiner to obtain boltz_yaml and boltz_files, then connect those here. Configure prediction parameters such as the number of steps, number of samples, and output format, then execute to obtain 3D structures and associated confidence/affinity data suitable for visualization and downstream analysis.

## Important Notes
- **Note 1**: The boltz_yaml and boltz_files inputs must be consistent; any file paths or identifiers referenced in the YAML must exist in the files bundle or prediction will fail.
- **Note 2**: Affinity outputs are only produced if the YAML includes an appropriate properties block (for example, specifying the binder chain) and affinity prediction is enabled.
- **Note 3**: Increasing steps or num_samples can substantially increase runtime and memory requirements; start with modest values and scale up as needed.
- **Note 4**: PAE, PDE, and pLDDT outputs are raw numeric arrays intended for quantitative analysis or visualization; they are not directly human-readable without plotting or specialized tools.
- **Note 5**: Ensure the chosen output_format matches what your downstream visualization or analysis software expects to avoid file-compatibility issues.

## Troubleshooting
- **Issue 1 – Prediction fails with missing file or reference errors**: Check that every file or identifier mentioned in boltz_yaml is present in boltz_files and that names, paths, and case sensitivity match exactly.
- **Issue 2 – No affinity_predictions output is produced**: Confirm that the YAML contains an affinity-related property configuration (for example, with a defined binder chain) and that affinity_prediction is enabled. Rebuild properties using the appropriate Boltz property builder if necessary.
- **Issue 3 – Very low confidence or unstable structures**: Reduce problem complexity (fewer chains or residues), verify that sequences and templates are correct, and consider increasing steps or num_samples to improve model quality.
- **Issue 4 – Structure files cannot be opened in downstream tools**: Verify that output_format is set to a supported value (such as pdb or mmcif) and that your downstream application is configured to read that format.
- **Issue 5 – Node runs out of resources or times out**: Lower num_samples, reduce steps, or simplify the system (shorter chains, fewer templates, fewer constraints) to reduce compute and memory demands.
