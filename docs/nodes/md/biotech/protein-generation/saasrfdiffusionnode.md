# RF Diffusion

<div style="display: flex; gap: 20px; align-items: flex-start; margin-bottom: 20px;">
<div style="flex: 1; min-width: 0;">

RF Diffusion runs the RFdiffusion model as a managed Salt SaaS service to generate new protein backbones. It supports unconditional de novo design, conditional design from an input PDB (including partial diffusion), and advanced modes via symmetry, contig mapping, guiding potentials, and scaffold-guided/fold-conditioned configuration. The node validates your setup, builds a structured RFdiffusion config, calls a GPU backend, and returns the designed structure plus optional diffusion trajectories.

## Usage

Use this node when you want to design or remodel protein backbones using RFdiffusion inside a Salt biotech workflow. In unconditional mode, leave input_pdb empty and specify length-only contigs (e.g. "100-100") to generate new monomers/oligomers from scratch. In conditional mode, provide an input_pdb and motif-style contigs (e.g. including chain letters and fixed ranges) for motif scaffolding, redesign, or binder design; partial_T<50 then enables partial diffusion from a noisy version of your input. For more specialized control, chain it with RF Diffusion Symmetry Config (for cyclic/dihedral/point-group symmetry), RF Diffusion Contigmap Config (for inpainting and length control), RF Diffusion Denoiser Config (noise schedules), RF Diffusion Potentials Config (guiding potentials), and RF Diffusion Scaffold Guided Config (fold-conditioned/scaffold-guided design using .pt files). Upstream you typically load and preprocess PDBs, scaffolds, and hotspot specifications; downstream you run structure prediction/ranking, property evaluation, and visualization/export. Use PROD for real designs, TEST for fast validation of parameter choices (short schedule, no trajectory), and MOCK only for UI or pipeline development where you need deterministic sample outputs without calling the backend.

## Important Notes
- **Performance**: RF Diffusion calls a GPU-heavy SaaS service; jobs with long contigs, many chains, or trajectory output can take minutes. Increase timeout accordingly and consider TEST mode to iterate on setup before long PROD runs.
- **Limitations**: In unconditional mode (no input_pdb), contigs must be pure length ranges (no chain letters) and hotspot_res must be empty; any violation will cause the node to raise a validation error before contacting the backend.
- **Behavior**: partial_T=50 is treated as no partial diffusion (internally converted to None). Any partial_T<50 requires an input_pdb and changes the diffuser configuration to perform partial denoising from a noised version of the input.
- **Behavior**: TEST mode overrides some inputs: it sets final_step to 49, disables cautious mode, and forces write_trajectory to false to speed up execution. Use PROD for final designs once parameters are validated.
- **Limitations**: hotspot_res in scaffold-guided workflows can only be used when a target protein is configured in the Scaffold Guided Config; otherwise, the node raises an error explaining that hotspots must refer to residues on a target protein.
- **Behavior**: Outputs are wrapped with metadata (including rf_diffusion config and per-output seeds) via the AlphaFoldBaseNode helper, which downstream nodes can use for reproducibility, tracking, and organized storage.

## Troubleshooting
- **Common Error 1**: "Non empty `contigs` must be passed" – The node requires contigs unless scaffold-guided mode is configured to auto-handle them. Ensure you supplied a non-empty contigs string or a valid Scaffold Guided Config that sets scaffoldguided to true.
- **Common Error 2**: "For unconditional sampling, expected `contigs` to specify only length range" – You used chain letters or motif-style segments without providing input_pdb. Either add an input_pdb for conditional mode or rewrite contigs as length-only (e.g. "150-150").
- **Common Error 3**: "For partial diffusion sampling, expected `input_pdb` to be provided" – partial_T is set below 50 but input_pdb is missing. Add a PDB for redesign/partial-diffusion or restore partial_T to 50 for full diffusion.
- **Common Error 4**: "hotspot_res=... was specified, but no target protein is configured" – In scaffold-guided mode, you set hotspot_res without specifying a target protein in Scaffold Guided Config. Configure target_pdb (and optionally target_pt_files) or clear hotspot_res.
- **Common Error 5**: Job times out or hangs – Increase timeout, reduce final_step, disable write_trajectory, or simplify your design (shorter contigs, fewer chains). For debugging connectivity or validation issues, run in TEST mode first to confirm the configuration executes successfully.
